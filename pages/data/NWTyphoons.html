An Analysis of
NW Pacific Typhoons

An interactive visualization showing a detailed analysis of typhoons in the NW Pacific Region (the most typhoon-active region).

View Dashboard in Tableau Public

Technical Summary

  • Data Source: IBTrACS, Wikipedia
  • Analysis Tool: Python, MS Excel
  • Data Viz Tool: Tableau Public 2023.3
  • Link: An Analysis of Typhoons in the NW Pacific Basin

Overview

Introduction

Typhoons, Cyclones, Hurricanes..
However you call it, they will always be a threat to livelihood and human lives. The NW Pacific Basin produces on average 26 storms per year making it the most active region for typhoons.
In this dashboard I created, Let's explore all the storms that have ever hit this area. We will try to identify patterns in terms of storm frequency, pressure and the destruction they have caused.

Scope

This dashboard includes all storm tracking data from 1980 to 2022. Only storms that were formed on the Northwestern Pacific basin are included. To be specific, the NW Pacific basin is the area in the Northern Hemisphere (Latitude > 0°) and with Latitude between 100° - 180°.

How to use the dashboard

The dashboard is divided into 3 different views: By Year, Patterns, and Impacts

By Year

The By Year view displays a map showing typhoon tracking data and strength for a specified Pacific Typhoon Season. The typhoon season is selected on a dropdown on the upper-left side of the map, below which contains the names of all storms that hit that year. Each name, when clicked, will display the track of ONLY the selected storm.

On the left sidebar, more details can be seen about the selected Typhoon season/year. Number of Storms, Max Storm Strength and Lowest Pressure are included as well as a ranked bar chart of strongest typhoons that year. A monthly breakdown is also included to show # of typhoons per month.

The main area contains a map which shows the track of all the storms on the selected typhoon season, OR a selected storm name. Each dot on the track represents the location of the storm per day and is colored according to the storm's strength. Each storm path is also labeled with the storm's name. When cursor is hovered on one of the dots, the speed of storm (in knots) at said location is also displayed.

At the bottom of the map, is an overall breakdown of # of typhoons per typhoon season/year.

Patterns

The 'Patterns' view shows each of the following:

  • How many expect per Month: # of storms that hit per month. It is further breakdown according to category (based on Saffir-Simpson scale)
  • Where do they form?: A map highlighting the areas according on the # of storms that have formed.
  • Pressure to Wind Speed Correlation: A scatter plot showing correlation between Pressure and Wind Speed
  • Throughout the Years: A bar chart showing number of typhoons per year and further breakdown by the maximum category of each storm (based on Saffir-Simpson Scale). It is overlayed with a line chart showing the max wind speed reached per year.
  • Areas likely to be hit: shows 4 heatmaps (one for each quarter) showing areas that have been within range of a storm.

Implementation

Data Sources

Storm Meteorological Data

The data used in this dashboard is obtained from the International Best Track Archive for Climate Stewardship (IBTrACS) project. Their website contains the most complete global collection of tropical storms available. Also, it's completely available to the public.

For this dashboard, I used the IBTrACS Version 4 which was their latest version.

Storm Impact

Although, the IBTrACS data is large dataset, it still does not have data for the # of deaths and damages each storm caused. After a little bit of research, I found many sources to have different numbers and most of these numbers are not in an organized form and rather, they are nested within articles which makes it very inefficient to look into each storm on record and read all the articles related to each. I expected each countries government agencies to have the data but I can't seem to find organized data form their websites.

An efficient, though unreliable solution would be to use Wikipedia. Suprisingly, they have an article for each typhoon season which include countries affected, # of deaths and resulting damages in USD. I decided then to utilize it for the lack of better data source. Data Extraction and Transformation

IBTrACS Dataset

The IBTrACS data can be downloaded on a csv format which I preferred since it's easier to manage. The fields used in the dataset are described in their Column Documentation. The dataset is opened with MS Excel. Data is then filtered to only include rows whose BASIN field has a value of WP which represents Western North Pacific.

Damages and Deaths Data from Wikipedia

To minimize effort in going through each wikipedia article, I created a script which automates extraction of data. The script iterates through each year from 1980 to 2022 and goes to the URL of each article determined by the pattern: https://en.wikipedia.org/wiki/{YEAR}_Pacific_typhoon_season#Season_effects. The script was left to run and finished within 5 mins.

The resulting data was exported to an excel file. The data is then cleaned and transformed so that each row represents one storm-area combination. It is further transformed to include a new column which identifies the country of each area. Example: data from wikipedia includes 'Bonin Islands' as an affected area by Typhoon Malakas in 2022. By looking it up, the area is identified to be within jurisdiction of Japan. Thus, all 'Bonin Islands' will have the value 'Japan' in the country field.
